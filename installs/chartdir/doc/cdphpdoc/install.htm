ChartDirector Ver 3.0 (PHP Edition)

Installation

Windows Installation

This section describes how to install ChartDirector for PHP under Windows. For the Linux, FreeBSD or Solaris version, please refer to Linux/FreeBSD/Solaris Installation.

PHP Version Compatibility on Windows

ChartDirector for PHP on Windows requires at least PHP versions 4.0.4pl1, and has been tested up to PHP Ver 4.3.1 (the latest PHP release as of the date of this document).

Installing ChartDirector for PHP on Windows

ChartDirector for PHP on Windows is distributed as a zip file. To install ChartDirector:

  • Unzip the ChartDirector distribution.

  • The ChartDirector for PHP module and its supporting DLLs are in the "ChartDirector\lib" subdirectory. Copy all files in the "ChartDirector\lib" subdirectory to the PHP extension directory in your system.

    The PHP extension directory is determined by the "extension_dir" variable in your PHP configuration file. The PHP configuration file is usually called "php.ini" and is in the [windows] directory.

    If you are not sure where is your PHP configuration file, create a web page with just one line:

    <?php echo phpinfo() ?>

    Then access this page with your browser. It will tell you the file name and location of your PHP configuration file.

    If the "extension_dir" variable is configured as a relative path name, please make sure you know where it is relative to (it depends on how your PHP is compiled, the web server you are using, etc). To avoid confusion, you may want to modify "extension_dir" to an absolute path name.

  • ChartDirector will dynamically load the ChartDirector PHP extension when needed. However, in some PHP systems, dynamic loading of extensions may be disabled. If this is the case, you would need to edit your "php.ini" file to configure PHP to pre-load ChartDirector on start up.

    The followings are two common cases in which dynamic loading of PHP extensions is disable:

    • Your PHP system may be configured to run in "safe mode" or with "enable_dl = Off" (configured in the PHP configuration file). In these cases, dynamic loading of extensions is disabled.

    • For PHP running as a web server module (eg. Apache module or ISAPI, as opposed to CGI) on Windows, dynamic loading of extensions is usually disabled. It is because PHP web server modules on Windows are usually multi-threaded, and PHP does not support dynamic loading of extensions in multi-thread web server modules.

    If your system cannot use dynamic loading of PHP extensions, you would need to add one of the following line to your PHP configuration file ("php.ini") to pre-load ChartDirector on start up.

    PHP VersionStatement
    4.0.4pl1extension=phpchartdir404.dll
    4.0.5 - 4.0.6extension=phpchartdir405.dll
    4.1.0 - 4.2.0extension=phpchartdir410.dll
    4.2.1 and aboveextension=phpchartdir421.dll

    If you are running PHP as a web server module, you would need to restart the web server for the extension statement to take effect.

Installing Sample Programs on Windows

ChartDirector for PHP comes with a number of sample PHP scripts under the "ChartDirector\phpdemo" directory. They are good examples and tutorials on how to use ChartDirector.

You may run these sample scripts to verify that ChartDirector is correctly installed.

To view the sample PHP scripts, simply copy them to your web server directory. Then access the "index.php" page using a browser.

If ChartDirector is installed correctly, you should see sample charts being generated by the sample PHP programs as you click through the links in the "index.php" page.

If for some reason, you cannot see the charts, please click on the "check installation" link on the "index.php" page. This will show the error messages, which may be useful to diagnose the problem.

Installing ChartDirector License on Windows

If you have purchased a license to use ChartDirector, you should have a license code delivered to you via email and postal mail.

To install the license code, follow the steps below:

  • Save the license code in an ASCII text file, and name the file "chartdir.lic". The ASCII text file should contain just one line, which is the license code. A sample license file is available at http://www.advsofteng.com/chartdir.lic for your reference.

  • Put the license file in the same directory as "chartdir.dll", which should be in your PHP extension directory.

  • Make sure the web server "anonymous user" has sufficient privileges to read the license file.

Using ChartDirector for PHP on Windows

Any PHP script that uses ChartDirector need to include the ChartDirector script "phpchartdir.php". Please copy the "phpchartdir.php" (located in the "ChartDirector\lib" subdirectory) to your PHP scripts directory. In your PHP scripts, please include the following statement:

require_once("phpchartdir.php");

You may also put the "phpchartdir.php" in other subdirectory. In this case, you need to include the path name in the "require_once" statement. Please refer to the PHP documentation for details on how to use "require_once".

Linux/FreeBSD/Solaris Installation

This section describes how to install ChartDirector for PHP under Linux/FreeBSD/Solaris. For the Windows version, please refer to Windows Installation.

PHP Version Compatibility on Linux/FreeBSD/Solaris

ChartDirector for PHP on Windows requires at least PHP versions 4.0.4pl1, and has been tested up to PHP Ver 4.3.1 (the latest PHP release as of the date of this document).

Installing ChartDirector for PHP on Linux/FreeBSD/Solaris

ChartDirector for PHP on Linux/FreeBSD/Solaris is distributed as a tar.gz file. To install ChartDirector:

  • Extract the files from the tar.gz file using:

    gunzip [chartdir_file_name].tar.gz tar xvf [chartdir_file_name].tar

  • The ChartDirector for PHP module and its supporting shared objects and fonts are in the "ChartDirector/lib" subdirectory. Copy *everything* (recrusive copy) in "ChartDirector/lib" (including the subdirectories inside) to the PHP extension directory in your system. This is by first changing to the "ChartDirector" directory, then use:

    cp -R lib/* [php_ext_dir]

    where [php_ext_dir] is the PHP extension directory in your system.

    The PHP extension directory is determined by the "extension_dir" variable in your PHP configuration file. The PHP configuration file is usually called "php.ini". It could be in various directory depending on how your PHP is compiled.

    If you are not sure where is your PHP configuration file, create a web page with just one line:

    <?php echo phpinfo() ?>

    Then access this page with your browser. It will tell you the file name and location of your PHP configuration file.

    If the "extension_dir" variable is configured as a relative path name, please make sure you know where it is relative to (it depends on how your PHP is compiled, the web server you are using, etc). To avoid confusion, you may want to modify "extension_dir" to an absolute path name.

  • ChartDirector will dynamically load the ChartDirector PHP extension when needed. However, in some PHP systems, dynamic loading of extensions may be disabled. If this is the case, you would need to edit your "php.ini" file to configure PHP to pre-load ChartDirector on start up.

    The followings are two common cases in which dynamic loading of PHP extensions is disable:

    • Your PHP system may be configured to run in "safe mode" or with "enable_dl = Off" (configured in the PHP configuration file). In these cases, dynamic loading of extensions is disabled.

    • For PHP running as a multi-threading web server module, dynamic loading of extensions is disabled. It is because PHP does not support dynamic loading of extensions in multi-threading web server modules. Note that most PHP modules on Linux/FreeBSD/Solaris are single-threaded.

    If your system cannot use dynamic loading of PHP extensions, you would need to add one of the following line to your PHP configuration file ("php.ini") to pre-load ChartDirector on start up.

    For single-threaded PHP
    PHP VersionStatement
    4.0.4pl1extension=phpchartdir404.dll
    4.0.5 - 4.0.6extension=phpchartdir405.dll
    4.1.0 - 4.2.0extension=phpchartdir410.dll
    4.2.1 and aboveextension=phpchartdir421.dll

    For multi-threaded PHP
    PHP VersionStatement
    4.0.4pl1extension=phpchartdir404mt.dll
    4.0.5 - 4.0.6extension=phpchartdir405mt.dll
    4.1.0 - 4.2.0extension=phpchartdir410mt.dll
    4.2.1 and aboveextension=phpchartdir421mt.dll

    If you are running PHP as a web server module, you would need to restart the web server for the extension statement to take effect.

ChartDirector for PHP comes with a number of sample PHP scripts under the "ChartDirector/phpdemo" directory. They are good examples and tutorials on how to use ChartDirector.

You may run these sample scripts to verify that ChartDirector is correctly installed.

To view the sample PHP scripts, simply copy them to your web server directory. Then access the "index.php" page using a browser.

If ChartDirector is installed correctly, you should see sample charts being generated by the sample PHP programs as you click through the links in the "index.php" page.

Note that the sample PHP scripts assume you are running PHP as an Apache module or web server module. If you are running PHP as a CGI, you may need to modify the sample scripts by adding "#!{path_to_php_interpreter}" to the first line of ".php" file.

If for some reason, you cannot see the charts, please click on the "check installation" link on the "index.php" page. This will show the error messages, which may be useful to diagnose the problem.
